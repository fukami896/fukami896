在Ubuntu上安裝PyCharm

最近剛開始學Python，看了A Byte of Python，寫得很簡單易懂，裡面推薦了PyCharm這套IDE，想想我也很久沒裝IDE這種東西了，一直都是用筆記本這種小程式，雖然乾淨，不過對於寫程式來說，還是不太方便，所以就在公司的電腦上裝了用用看。沒想到，還真的挺好用的，執行寫好的程式只要點一下就可以在下方看到結果，比起切換視窗打CLI快多了。所以回家就來裝個PyCharm在我的Ubuntu 14.04上，不過官方沒有提供apt-get整包安裝，所以從網路上找了其它方法，我想這大概是好心人提供的.deb檔吧。感謝大大w

以下為安裝方法：

wget -q -O – http://archive.getdeb.net/getdeb-archive.key | sudo apt-key add –
sudo sh -c ‘echo “deb http://archive.getdeb.net/ubuntu $(lsb_release -sc)-getdeb apps” >> /etc/apt/sources.list.d/getdeb.list’
sudo apt-get update
sudo apt-get install pycharm

就這簡單的四行，別忘了裝完後到Setting裡去調一下要用的Python Interpreter版本啊
之後再來寫一下不錯的學Python網站

reference: Installing PyCharm 3.1 on Ubuntu 14.04

Ubuntu上的行列輸入法

我大概是在兩年前的夏天開始用行列輸入法的，其實也沒什麼特別的原因，就只是在打工時被新注音給氣了太多次(那時我是在打逐字稿)，所以才終於下定決心開始學其它的輸入法。

接著在挑輸入法時，我也沒有想太久，就決定用行列，因為我想要打完就不用再選字的輸入法(重碼沒有或是很少)，又想要是Windows有內建的，所以就選了之前就有點興趣的行列來學。

到了現在大概一年多的時間，大概是因為我沒有打字打得很勤的關係吧，我的打字速度還是沒有多快，但是至少在打字時我不用一直移動我的右手去選上下鍵，光是這點我就已經覺得很開心了。

可是在我換到Ubuntu後又出現了問題，因為我一直沒找到好用的行列30輸入法，去官方的iBus wiki上點撃安裝的行列30，也是一個爛得可以的東西，選字順序完全和Windows上的不一樣，有事沒事就一直打錯字，真的很煩燥。還好，我剛剛查了一下，終於發現還有另一個行列，一樣可以用apt-get安裝，而且用起來還跟Windows上一樣，這真是天大的福音啊！

總之，這個套件包叫做ibus-array，打開Terminal輸入：

sudo apt-get install ibus-array

就好了

記住，不要像Wiki上一樣裝這個ibus-table-array30，會讓你氣到瘋掉XDD

ibus-array官網

搭建Hexo過程中的注意事項

1. 按照官網寫的一步步安裝完成，重開terminal後執行hexo卻說command not found

Solution: 雖然不知道為什麼，不過似乎跟沒有指定nodejs的default版本有關，所以要在terminal中輸入以下指令確認nvm版本:

$ nvm ls
接著再指定default的nvm版本: 
$ nvm alias default [nvm 版本號]

這樣就解決了
reference:HexoでGitHubにブログをつくる

2. 幫Hexo加上RSS功能

Hexo有很多Plugin可以用，RSS feed就是其中一個
只要打幾個指令就能安裝了

首先打開Terminal進到blog資料夾，輸入：

$ sudo npm install hexo-generator-feed —save

然後打開_config.yml，在裡面加上：

#Feed Atom
feed:
    type: atom
    path: atom.xml
    limit: 20

接著只要generate一次之後，就可以在blog資料夾下發現多了atom.xml，增加RSS feed完成！

3. 增加留言功能

這個留言功能是使用Disqus的服務來達成的，所以首先要到Disqus申請一個帳號，並且在其中增加自己的網站網址

接著打開_config.yml，找到disqus_shortname，填上自己的shortname就大功告成了(其實這裡我也不太懂，大概是填入帳號名吧，總之我是成功了XDD)
reference: Hexo 安裝教學、心得筆記 (Windows)

4. Push到Github時使用SSH，免輸入帳密

因為我懶得每次更新blog時都要輸入一次帳密，所以想來研究一下有沒有比較方便的方法，剛好Github也能讓你使用SSH來Push，所以就來用用看，只是要先做些前置準備

詳細步驟請先看Github官方教學Generating SSH keys
按照步驟做完後，就可以在自己的Github repository頁面右方看到可複製的SSH連結，將他貼到_config.yml中的repository後，就可以用啦~

在Ubuntu上看漫畫

自從開始用Ubuntu之後，其實我沒什麼不滿的，因為大部分的軟體都找得到Linux版的代用品，不然就是有Linux版可用。但是，有幾個真的找不到的，一個是foobar2000，另一個則是我常用的看圖軟體— Hamana。

為什麼這樣說呢，因為在我試用過了幾個網路上推薦的看漫畫or看圖程式— Mcomix、Comix、Comicseer和Geeqie之後，我發現，其實在看圖這件事上，所有的軟體其實都差不多，也大都可以看壓縮檔(Geeqie不行)，但是，在目錄間移動&看縮圖這件事上，不是做半套、不方便，不然就是根本沒做這功能，使得我根本沒辦法專心看載好的本本，實在是令人煩悶到一個極點。

所以我只好回頭來挖出以前在Windows上慣用的Hamana，用幾乎放棄的心情，使用Wine來跑跑看，結果令人意外的事情發生了! Hamana跑得好好的，除了字體有點醜之外，其他的功能都完好無缺，之前的設定檔也都留著，讓我可以馬上回到以前方便看漫畫的狀態，再也不用忍受Linux垃圾般的看漫畫軟體，這真是太棒了啊!!!

結論︰想用Linux原生的看漫畫程式? 放棄吧，用Wine跑你習慣的程式比較快(拇指

同場加映：
萬用模糊技法(Pixiv)，簡述如下：

  1. 弄個圖層0拷貝，放在0上面，模式為色彩增值
  2. Ctrl+A全選畫面，編輯>拷貝合併&貼上，模式為濾色，隨便高斯模糊一下
  3. 再弄個0拷貝，放在最上面，模式為變亮 —> 完成

真的隨便都超有質感，有沒有這麼爽www
